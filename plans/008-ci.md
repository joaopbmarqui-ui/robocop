## Why this matters

There is no CI on either remote (`.github/workflows/`, `bitbucket-pipelines.yml`
both absent). Regressions land undetected before Bitbucket snapshot publish or
edge deploy. The repo has two remotes (`origin` = GitHub for issues, `bitbucket`
= corporate writable remote per `AGENTS.md:121-123`); GitHub Actions is the
reachable gate from a Cursor Cloud/dev context and covers PRs pushed to
`origin`. This plan adds a single GitHub Actions workflow that runs
compileall, ruff, mypy, and pytest on push and PR, so a regression is caught
before it reaches either remote.

## Steps

### Step 1: Create the workflow file

Create `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false
      matrix:
        python-version: ["3.10", "3.12"]
    steps:
      - uses: actions/checkout@v4
      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
      - name: Install dev dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -e ".[dev]"
      - name: Compile
        run: python -m compileall dispatch scr
      - name: Lint
        run: ruff check dispatch tests
      - name: Format check
        run: ruff format --check dispatch tests
      - name: Typecheck
        run: mypy dispatch
      - name: Help smoke
        run: python -m dispatch --help
      - name: Tests
        run: python -m pytest tests tools/prod_tui/tests -q
```

If Plan 007 has not landed yet, the `Lint`/`Format check`/`Typecheck` steps
will fail because `ruff`/`mypy` are not installed and not configured. This
plan **depends on 007** — do not run 008 before 007.
