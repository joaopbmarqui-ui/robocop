## Direction findings not planned by default

- **Resume partial `Table+Csv` Jobs**: ADR-0003 names a future resume feature
  for failed CSV export after table creation. This is valuable but should
  follow Plans 002-005 because it extends the manifest/runner contract.
- **Browser `ExistingTable -> Csv` export shortcut**: The Browser can describe
  and drop tables, while `ExistingTable -> Csv` is already legal. This is a
  good small product feature after Plan 001 validators exist.
- **App-wide completion notifications**: Dashboard owns transition toasts today.
  Lift them to the app after Plan 004 defines the polling model.
- **Post-v1.0 `scr/` consolidation**: ADR-0005 explicitly allows
  consolidation and dead `--download` removal, but Plan 005 should land first.
- **SSH deploy keys and staging-cluster tier**: Useful operational hardening,
  but require operator/infra choices outside source-only execution.

## Findings considered and rejected

- `StrictHostKeyChecking=no` in the production harness: documented in the edge
  operating model; not a source-code fix without changing operator SSH policy.
- Mock environment shelling out to fake Hadoop/Kerberos/SMTP tools: intentional
  ADR-0004 design, not a security finding.
- `MAILHOST` and `DISPATCH_SCR_DIR` environment overrides: ADR-0005 allows
  environment externalization for mocks; hardening can be revisited if a
  production profile flag is introduced.
- Immediate plan for unbounded `scr/` retry loops: real risk, but ADR-0005
  makes behavior changes production-sensitive; first land Plan 005 coverage.
- Immediate plan for monthly cross-coordinator `TABLE_NOT_FOUND`: ADR-0005
  documents attempted fixes and says the correct fix needs structural
  cluster/operator input, so this audit does not hand it to a generic executor.
- Version-string drift (`VERSION`/`dispatch/version.py` 1.1.0 vs
  `pyproject.toml` 1.0.0): valid small cleanup, but lower leverage than launch
  correctness and security boundaries.
- Missing Linux `local_check.sh` and CI workflow: valid DX work, but lower
  immediate product risk than the five selected plans.
